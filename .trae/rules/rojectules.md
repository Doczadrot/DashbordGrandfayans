Project: Quality Dashboard
1) Данные: основной вход — файлы (XLSX/CSV/TXT). Поля для аналитики: Номенклатура, Персональный номер, Причина претензии, Описание дефекта, Поставщик. 1 строка = 1 штука.
2) Нормализация: дефекты нормализуются машинно, но каждое автоматическое объединение требует подтверждения пользователя (poll).
3) UI-политика: дашборд — HTML/CSS/JS с iOS-стилистикой (см. UI_STYLE.md). Плавные переходы, крупная типографика, цветовая кодировка причин.
4) Ответственность: TRAE действует как консультант — все управленческие решения принимает пользователь.
5) ты логируешь каждое действие пользователя, включая изменения в данных, фильтры, сортировки, и другие действия интерфейса.
6) ты сохраняешь удачные версии интерфейса, сохраняя их в отдельный файл для будущих улучшений, а затем покрываешь их тестами
7) ты исключтельна рабоатешь по тем файлам котоыре треубеются для реализации задачи, другие файлам ты не трогай.
8) есщи запрос повторяется пользователем снова и снова -опровси его метородом опроса для того что бы лушче понять пользоваетля.
9) Ты должеш считать продажи по каждой позиции в файле данных, суммируя их стоимость. ( пример относитися только просчета продаж на МП)
